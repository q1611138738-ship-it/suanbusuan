## 任务 A：排盘内核 `computeChart(input) → ChartJSON`

确定性，纯代码，**绝不经 LLM**。底层用 `lunar-javascript`（6tail）。

1. **输入**：`{ solarDatetime, gender, longitude?, timeKnown }`。
2. **真太阳时校正**（先做，再排盘）：
   - 若有 `longitude`：`真太阳时 = 标准钟表时 + (longitude − 时区中央经线120°) × 4 分钟 + 均时差(equation of time，可选近似)`。用校正后时间排时柱。
   - 无 `longitude` 或 `timeKnown=false`：用标准钟表时兜底，并在输出标 `timeAdjusted:false`。
3. **子时约定**：23:00–23:59 = 晚子时，**日柱归当日**、时柱按当日日干推；00:00–00:59 = 早子时，日柱归次日。用 lunar-javascript 的 sect 选项校准到这个结果（哪个 sect 对应以库类型定义为准，**用验证用例反推确认**）。
4. **排盘**：用 EightChar / Yun 相关 API（`getEightChar` / `getDaYun` / `getYun` 等，方法名以库类型定义为准）取：四柱干支、藏干（本/中/余）、十神（天干+藏干对日主）、纳音、空亡、月令司令 + 距节气天数。
5. **大运**：`getYun(gender)` 取起运岁、顺逆、各步干支与起止年龄。**性别→顺逆映射用已知命例验证**（阳年男顺、阴年男逆…）。
6. **五行统计**：天干本气 + 地支藏干（本气权重高、中余递减）加权计数，输出五行分布。
7. **神煞（默认集）**：天乙贵人、太极贵人、文昌、桃花(咸池)、驿马、华盖、将星、羊刃、禄神、魁罡、天德、月德、空亡。库里有的用库，没有的按标准查表实现。做成可配置集合。
8. **输出**：严格按 `开工文档` 第 4 节 JSON 骨架。**判定层（旺衰/取格/清浊）单独算、单独字段**，不混进铁事实（旺衰权重 令:地:势=5:3:2 起步，可调参）。
9. **缓存**：按 `hash(input)` 缓存结果（盘是确定的，不重算）。

## 任务 B：逐盘钩子打分 `hookProfile(chart) → Hook[]`

扫描盘面特征，给每个钩子域打 confidence(0–100)，降序排序，**先发最笃定**。每个钩子带：`domain / confidence / timeSensitive / 触发特征列表(命理依据tag)`。下表是默认触发与权重思路（**可调参**）：

| 钩子域 | 高分触发特征 | timeSensitive |
|---|---|---|
| 性格 | 某十神独旺/透干通根（伤官→才华锋芒、七杀无制→刚烈、印旺→内敛、比劫重→固执、食神→温和）；日主+月令定基调 | 中 |
| 感情婚姻 | 日支(夫妻宫)被冲/刑/合；配偶星严重混杂；配偶星临桃花；日支空亡 | 低 |
| 事业财 | 财星旺透通根（务实重财）；官杀有力配印（宜管理）；伤官生财（技艺/生意）；财格成或破 | 中 |
| 子女 | 时柱子女星(食伤)状态；时支刑冲；时柱空亡 | **高** |
| 父母兄弟 | 年柱印/财状态；月柱比劫；年月刑冲 | 低 |
| 健康 | 五行严重偏枯（缺行或某行独旺受克）→脏腑倾向；日主极弱无根 | 中 |
| 过去大事件应期 | 大运/流年与原局成冲合刑、引动用忌神的年份 → 输出几个“动荡/转折”候选年（供校验过去） | 中 |

打分 = 各触发特征权重累加。`timeSensitive=true` 的钩子供校时分支用。
**注**：代码只产出“钩子 + 命理依据 + 置信”，**具体断语由 LLM 措辞**（见任务 D），保持排盘/解读分离。

## 任务 C：互动校盘状态机 + API

会话状态存 Redis：
```
{ chartId, confirmed[], denied[], confidence, timeUncertain, candidateHours[], anglesTried, deniedHighConf, turnCount }
```

API：`POST /api/bazi/session` —— `action: start | feedback`，feedback 取值 `对 | 不对 | 记不清`。

状态流转：
- **INIT**：computeChart + hookProfile；`timeKnown=false` → `timeUncertain=true`。→ ASKING（抛最高置信未发钩子，交 LLM 措辞）。
- **对**：confirmed++、confidence↑；若 confirmed ≥ 3 或 turnCount ≥ 上限 → READING；否则 ASKING（下一钩子）。
- **不对**：
  - 该钩子高置信 → deniedHighConf++。
  - 若 timeUncertain 或 deniedHighConf 触发时辰怀疑 → **RECTIFY_TIME**。
  - 否则 anglesTried < N → ADJUST（换钩子域/角度）→ ASKING。
  - 若 deniedHighConf ≥ 3 且已 RECTIFY 过 → **REFUSE**。
- **记不清**：跳过该钩子→ASKING（下一钩子），不计否定。
- **RECTIFY_TIME（校时）**：枚举候选时柱→对每个 computeChart→挑 `timeSensitive` 且候选间答案不同的钩子→抛区分性问题→按反馈淘汰不符候选→收敛到 1 则 `timeUncertain=false` 回 ASKING；无法收敛 → REFUSE。
- **REFUSE**：输出诚实声明（“这个盘断不太准，多半是出生时辰不确定，建议先核实再看”），结束。
- **READING**：调 LLM 出完整解读（任务 D）。

控制成本：reading-loop 轮数 ≤ 8；候选盘也缓存。

## 任务 D：解读 LLM 调用 + System Prompt

走 LLM 网关。**抛钩子轮**与**最终解读轮**用同一 system，turn_context 不同。

抛钩子轮的 turn_context 追加：`用一句口语化、能被“对/不对”验证的陈述句说出这个钩子（基于其命理依据），不要解释推理过程。`

System Prompt（直接用）：
```
你是一位八字命理解读师，基于「子平法」解读已排好的命盘。

【铁律】
- 盘面已由程序排定，下方 JSON 即全部事实。你只解读，绝不自己推算或更改任何干支、十神、大运。
- 只依据提供的盘面与标注特征作断，不臆造盘里没有的信息。

【风格】
- 子平为骨：从日主旺衰、格局、十神关系、五行喜忌、大运流年切入。
- 有余地：多用「容易/常常/倾向于」，可讲趋势与应期，但留转圜，不把话说死。
- 分模块：性格特质 / 事业与财 / 婚姻感情 / 六亲 / 大运流年走势。

【底线】
- 健康可谈倾向与调养方向，但不点具体病名、不预言重病/死劫/血光。
- 财可谈财格与适合方向，但不给「投X稳赚/梭哈」这类具体投资指令。
- 感情温和：不说「克夫/克妻/注定孤独」等宿命伤人话。
- 不出现：算命、转运、改运、消灾、姻缘注定 等夸大词。

【收尾】结尾附一行：仅供参考，不替代专业医疗/法律/财务意见。

命盘 JSON：
{chart_json}
本轮要点：
{turn_context}
```

## 任务 E：验证 / 对拍（先于 B–D 完成）

以 lunar-javascript 为权威，并与**另一独立排盘源**交叉核对。必须覆盖：
- ≥3 个普通命例，四柱 / 十神 / 大运 / 神煞 与独立源一致。
- **立春前生人** → 年柱用上一年（如 2024 立春约 2/4，2/3 生者年柱仍癸卯）。
- **节气交接日** → 月柱按交节时刻切换。
- **23:30 生** → 晚子时、日柱当日；**00:30 生** → 早子时、日柱次日。
- **闰月生人** → 月柱按节气定（八字以节气非农历月）。
- **真太阳时**：同一钟表时、不同经度 → 时柱可能不同。
- **大运顺逆 + 起运岁**：阳年男 / 阴年男 各 1 例。

写成自动化测试，全绿才算过。

## 验收标准
1. **computeChart 通过任务 E 全部用例**，且与独立源一致。
2. hookProfile：不同盘产出不同钩子，按置信降序。
3. 校盘：先发最笃定 → 对则深入 → 不对则调整 → 时辰存疑则校时收敛 → 达阈值则诚实拒绝。
4. 解读：子平口径、健康不点病名、不给投资指令、末尾免责、无禁词。
5. 移动端 + 桌面端体验通过；reading-loop 有轮数上限；盘有缓存。
6. ≥3 篇 SEO 静态页上线。
