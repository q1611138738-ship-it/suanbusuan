## 0. 项目定位

- 国学 / 术数在线工具站，AI 辅助解读。**首发八字**，其余 5 个板块先灰度（敬请期待）。
- 部署在**境外**（台湾 VPS + Cloudflare），不走大陆备案。域名 `suanbusuan.com`。
- 变现走海外渠道（打赏 / 会员 / USDT），后续接入，v1 不做。
- **最高原则：排盘交代码（确定性，绝不让 LLM 计算盘面），解读交 AI。**

## 1. 技术栈（已定，勿改）

| 层 | 选型 |
|---|---|
| 前端 | Next.js (App Router) + TypeScript + Tailwind CSS + shadcn/ui |
| 后端 | Next.js API Routes（v1 前后端一体，预留拆分） |
| 缓存 / 会话 | Redis |
| 推理 | LLM 网关：DeepSeek 主 / Gemini 备（可切换） |

理由：Next.js 响应式开箱即用、组件化便于加板块、SSR 利于 SEO（大陆推广难，靠 SEO + 海外引流）。

## 2. 总体架构（分层）

```
┌─ UI 壳层（响应式，全板块共享）：导航 / 用户 / 支付 / 聊天容器 / 板块卡片
├─ 解读·对话引擎（共享框架）：互动校盘 + 逐盘钩子打分排序 + LLM 网关 + 会话状态
├─ 板块插件层（每个国学板块一个）
│     每个板块只提供三样 →【排盘函数】【钩子定义+打分规则】【解读 prompt 模板】
├─ 排盘内核（各板块各自实现，输出统一 JSON 骨架，确定性，不经 LLM）
└─ 配置层：板块注册表（6 板块 启用/灰度 开关）+ 文案/内容（与代码分离）
```

**核心：上面三层（UI 壳、对话引擎、网关支付）所有板块共享。加新板块时只写最下面「排盘 + 钩子 + prompt」三样，核心代码一行不动。**

## 3. 板块插件接口（扩展性核心）

每个板块实现统一接口：

```ts
interface DivinationModule {
  id: string;                 // 'bazi' | 'ziwei' | 'qimen' | 'liuyao' | 'meihua' | 'fengshui'
  name: string;               // 显示名
  enabled: boolean;           // 灰度开关：false = 显示“敬请期待”并锁定
  order: number;              // 卡片排序
  inputSchema: InputField[];  // 多形态输入定义（见下）
  computeChart(input): ChartJSON;                       // 排盘：确定性，调库，绝不走 LLM
  hookProfile(chart): Hook[];                           // 逐盘钩子打分 + 降序排序
  buildReadingPrompt(chart, session): PromptTemplate;   // 解读 prompt 模板
}
```

### 3.1 多形态输入（⚠️ 一开始就抽象，否则灰度转正要返工）

不同术数输入方式完全不同，输入采集层必须**可配置多形态**，不能写死成生辰表单：

| 板块 | 输入形态 |
|---|---|
| 八字 / 紫微 | 出生年月日时 + 性别 + 出生地（经度，用于真太阳时） |
| 奇门 / 风水（择时/方位） | 起局时间（默认当前，可指定）+（风水另加方位/坐向） |
| 六爻 | 摇卦（三枚硬币 ×6）/ 报数 / 时间起卦 |
| 梅花 | 报数 / 时间 / 输入字 |

`InputField` 抽象出：类型（date/time/select/coin/number/text/geo）、是否必填、默认值、兜底逻辑。

### 3.2 板块注册表（配置驱动灰度）

```ts
const MODULES = [
  { id:'bazi',     name:'八字',     enabled:true,  order:1 },
  { id:'ziwei',    name:'紫微斗数', enabled:false, order:2 },
  { id:'qimen',    name:'奇门遁甲', enabled:false, order:3 },
  { id:'liuyao',   name:'六爻',     enabled:false, order:4 },
  { id:'meihua',   name:'梅花易数', enabled:false, order:5 },
  { id:'fengshui', name:'风水',     enabled:false, order:6 },
];
```
首屏渲染 6 张卡片，`enabled:false` 的显示“敬请期待 / 即将上线”并锁定；可加“想要点这里”按钮收集需求。

## 4. 八字排盘内核 — 统一 JSON 骨架（铁事实层）

只放**客观事实**。旺衰、格局、用神、层次属“判定”，单独算、单独存（见 4.2），**不混进铁事实**。

```jsonc
{
  "input": { "solarDatetime":"...", "gender":"male|female", "longitude":121.5, "timeKnown":true },
  "pillars": {                       // 四柱
    "year":  { "gan":"甲", "zhi":"子" },
    "month": { "gan":"丙", "zhi":"寅" },
    "day":   { "gan":"戊", "zhi":"午" },
    "hour":  { "gan":"壬", "zhi":"戌" }
  },
  "hiddenStems": { "year":["癸"], "month":["甲","丙","戊"], "day":["丁","己"], "hour":["戊","辛","丁"] }, // 藏干(本/中/余)
  "tenGods": { "...对日主的十神(天干+藏干)..." : "..." },
  "fiveElements": { "wood":2, "fire":3, "earth":2, "metal":0, "water":1 },        // 五行加权统计
  "monthCommand": { "ruler":"甲", "daysFromTerm":7 },                              // 月令司令 + 距节气天数(深浅)
  "relations": { "he":[...], "chong":[...], "xing":[...], "hai":[...], "po":[...], "ganHe":[...] }, // 刑冲合会害+天干五合
  "shaSha": ["天乙贵人","桃花","驿马", "..."],                                      // 神煞(可配置集)
  "xunKong": ["子","丑"],                                                          // 空亡

  "luck": { "startAge":3.4, "direction":"forward", "pillars":[ { "gan":"丁","zhi":"卯","fromAge":3,"toAge":13 }, ... ] }, // 大运
  "currentYear": { "gan":"乙", "zhi":"巳", "year":2026 }                           // 流年
}
```

### 4.1 默认命理参数（我已替你定，写死为默认、可配置）

| 参数 | 默认值 |
|---|---|
| 子时划分 | 23:00–23:59 = 晚子时，**日柱归当日**、时柱按当日日干推；00:00–00:59 = 早子时，日柱归次日 |
| 真太阳时 | **默认按出生地经度校正**（影响时柱）；无出生地→用标准钟表时兜底，并标注“未校正，时柱可能有偏差” |
| 起运 | 3 日折 1 年，**精确到天** |
| 大运方向 | 阳男阴女顺排，阴男阳女逆排 |
| 神煞集（默认） | 天乙贵人、太极贵人、文昌、桃花(咸池)、驿马、华盖、将星、羊刃、禄神、魁罡、天德、月德、空亡（可增减） |

### 4.2 判定层（单独算，不进铁事实）

代码计算（尽量确定性，**不让 LLM 算**）：
- **旺衰评分**：权重 令 : 地 : 势 ≈ **5 : 3 : 2** 起步（可调参）→ 输出 身强/中和/身弱/从强/从弱 + 分值。
  > 注：v1 为简化模型，后续可替换更精算法。
- **取格**：八格（正官/七杀/正偏财/食神/伤官/正偏印）+ 建禄/阳刃 + 特殊格（从/化/专旺）。
- **格局清浊/成破/配合** → 层次评级（为 Phase 2 的“层次路由 B 架构”预留，v1 先不路由）。

## 5. 互动校盘 + 逐盘钩子（共享框架，本产品的灵魂）

### 5.1 逐盘钩子打分（不同盘用不同钩子，先发最笃定）

`hookProfile(chart)` 给每个钩子域打**置信分**，按置信降序排序，**先发最笃定的那个**。

高置信信号来源：强而专一的十神组合、宫位被明显刑冲合、格局极端、神煞扎堆、大运流年正引动某宫。模糊/矛盾/中和的信号不打头。

钩子分级（系统按此挑钩子，标签由 `hookProfile` 产出）：

| 钩子 | 验准确度 | 帮校时 | 风险 |
|---|---|---|---|
| 性格特质 | 强（常作首发） | 中 | 低 |
| 过去大事件应期 | 强（最好验） | 中 | 低 |
| 感情/婚姻状态 | 强 | 低 | 低（语气温和） |
| 子女情况 | 中 | **强**（时柱主子女） | 低 |
| 父母/兄弟 | 中 | 低 | 低 |
| 身体倾向 | 中 | 中 | **只输入不输出** |

### 5.2 互动状态机

```
session = { chartId, confirmed[], denied[], confidence, timeUncertain, candidateHours[], triedAngles[], turnCount }
```

流程：
1. `hookProfile` → 取置信最高且未发的钩子，发出（用陈述句抛“钩子”，如“你性子偏急、嘴上不饶人，对吗？”）。
2. 用户反馈：**对 / 不对 / 记不清**（按钮，零打字）。
3. **对** → 提高置信，顺该方向深入。
4. **不对** → 触发调整链：
   a. 先怀疑时辰 → 若时辰本就未知或存疑，进入**校时分支**（5.3）；
   b. 否则换断的角度/维度；
   c. 仍对不上 → 见拒绝阈值（5.4）。
5. 收尾 → 路由到细断 + 输出解读。

### 5.3 校时分支（高价值功能）

时辰未知/存疑时：内核排出候选时柱（12 时辰或可信子集），AI 专挑**不同时辰答案不同**的点提问（时柱敏感：子女/晚年/部分性格），靠用户回答逐步缩到正确时辰（贝叶斯式排除）。

### 5.4 拒绝阈值（合规护城河）

连续否定 **≥3 个高置信钩子**，且时辰已变换尝试过 → **不硬断**，诚实输出：
> “这个盘我断不太准，多半是出生时辰不确定。建议你先核实下出生的准确时辰，我们再看。”

## 6. 解读引擎 & 文案规则（八字 prompt 要求）

- **子平为骨**（性格 / 五行喜忌 / 人生节奏 / 大运大方向）；v1 先单派跑通，盲派与层次路由留 Phase 2。
- **倾向语气**：多用“容易 / 常常 / 倾向于”，给余地，少把话说死（说应期、趋势都行，但留转圜）。
- **健康 / 财运 / 事业 / 感情正常断**：按传统命理口径描述（五行→脏腑调养、财格→适合方向、官杀→事业等）。只守两条窄底线（也是保护自己——这两类最招投诉、诈骗指控与支付封号）：
  - ① 不点具体病名、不预言重病 / 死劫 / 血光；健康落到“留意 / 调养”。
  - ② 不给“投 X 稳赚 / 梭哈”式具体投资指令（可聊财格与适合方向）。
- **感情温和**：禁“克夫/克妻/注定孤独”等宿命伤人话术。
- **全程禁词**：算命、转运、改运、消灾、血光、姻缘注定 等夸大 / 恐吓词不出现在文案。
- **末尾免责**：仅供参考，不替代专业医疗 / 法律 / 财务意见。

## 7. 硬性护栏（不可妥协，代理不得跳过）

1. 排盘必须由代码完成，**LLM 绝不计算盘面**。
2. 健康 / 财守两条窄底线：不点具体病名、不预言重病 / 死劫 / 血光；不给“投 X 稳赚 / 梭哈”式具体投资指令（详见第 6 节）。
3. 末尾免责 + 禁词过滤；感情语气温和。
4. 隐私：生日属个人信息，写明用途、不滥存、提供隐私政策。

## 8. 自检要求（必须满足）

1. **SEO 静态页不能漏**：聊天内容动态生成爬不到，必须另做国学科普/板块介绍静态页（SSG）撑 SEO。Phase 1 至少 3 篇。
2. **多形态输入抽象**（见 3.1）：最大返工风险，一开始就做对。
3. **成本控制**：排盘结果缓存（盘确定，不重算）、限制单次互动轮数、prompt 精简。
4. **移动端 viewport**：聊天页处理输入法弹起、`100dvh`（非 `100vh`）抖动，保证手机体验。
5. **时辰未知 UX**：允许“不知道时辰也能开始”，用校时循环兜底；勿用必填时辰挡人。
6. **被墙备份**：配置支持快速切换备用域名 / IP。
7. **响应式**：移动端全屏沉浸聊天；桌面端居中列 + 板块侧栏。

### Phase 1 — 八字（上线）
- `computeChart`：基于 lunar-javascript，输出第 4 节统一 JSON。
- `hookProfile`：逐盘钩子打分 + 降序。
